### Best Practices

✅ **Do:**
- Include a direct link to your GitHub repository
- Share a screenshot of the UI in action
- Mention the specific technologies you used
- Explain the business value (time-saving, improved performance)
- Be genuine about what you learned

❌ **Don't:**
- Overclaim - be accurate about capabilities
- Hide your source code - show you're proud of it
- Forget to mention the tech stack
- Post without testing the application first
